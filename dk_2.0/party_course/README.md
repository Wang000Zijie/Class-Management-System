# party_course

党课管理系统（Flask + SQLite），围绕学期制培训场景实现学员、课程、签到、作业、志愿、成绩、证书和通知的全流程管理。

## 1. 项目定位

- 按学期组织党课培训，激活哪个学期，系统主界面就展示哪个学期的数据。
- 支持三阶段学员管理：积极分子、发展对象、预备党员。
- 支持按阶段配置独立“第几期”，并在成绩/作业/志愿等页面按期筛查。
- 内置 AI 智能同步入口，将学员、作业、考试、志愿更新合并到一个流程中。

## 2. 技术栈

- 后端：Flask, Flask-Login, Flask-SQLAlchemy
- 数据库：SQLite
- 表格导入导出：openpyxl
- 前端：Jinja2 + Bootstrap 5

依赖见 [requirements.txt](requirements.txt)。

## 3. 功能总览

### 3.1 学期管理

- 新建/编辑/复制/激活学期
- 学期级参数：通过线、志愿目标时长、权重配置等
- 三阶段独立期数：
	- 积极分子期数
	- 发展对象期数
	- 预备党员期数

### 3.2 仪表盘

- 展示当前激活学期统计：人数、课次进度、出勤率、通过情况
- 显示当前激活学期的三阶段期数
- 不在仪表盘修改期数，统一在学期管理维护

### 3.3 学员管理

- 新增、编辑、删除、导入、导出、分组
- 自动绑定到当前激活学期
- 同学号在同学期同阶段唯一，允许跨阶段并存
- 学员姓名左侧可展示“第X期（学期名）”标签（对应其阶段期数）

### 3.4 课程与签到

- 课程与签到管理已合并为同一业务模块入口
- 课程课次管理、详情、编辑、删除
- 签到状态管理与汇总
- 支持批量签到与资格取消相关流程

### 3.5 作业与志愿

- 作业：总览、明细、批量录入、汇总导出
- 志愿：总览、明细、批量录入、汇总导出
- 两个总览页面都支持“阶段 + 第几期”筛查

### 3.6 成绩与证书

- 成绩计算、结果总览、公示与导出
- 出勤门槛按规则判定（硬条件）
- 结果页支持“学期 + 阶段 + 第几期 + 总分排序（升序/降序）”筛查
- 打开结果总览时会自动重算当前筛选学期，避免 FinalScore 缓存滞后
- 若学员改动后从“达标”变为“不达标”，已发证书编号会自动追加“(错误)”标记；恢复达标会自动去除该标记
- 证书号按阶段规则生成（期数来自对应阶段期数字段）

### 3.7 通知模板

- 模板管理、编辑、使用与导出
- 支持分类管理与实际通知场景复用
- 干事仅可查看/使用模板，不能新增、编辑、删除

### 3.8 AI 智能同步

- 入口为“AI智能同步”
- 合并学员/作业/考试/志愿更新能力
- 有 API Key 时走 DeepSeek 解析，无 Key 时走本地兜底解析
- 专用提示词文件：
	- [ai_agent_prompts/smart_sync_prompt.txt](ai_agent_prompts/smart_sync_prompt.txt)

### 3.9 权限与日志

- 三层账号权限：主席（admin）、部长（minister）、干事（staff）
- 主席：最高权限，可查看日志、可新增/删除账号
- 部长：可查看日志，业务权限与主席一致，但不能管理账号
- 干事：不能查看日志、不能管理账号、不能编辑通知模板
- 所有写操作（POST/PUT/PATCH/DELETE）会自动记录到操作日志
- 日志页支持分页、筛选、导出 Excel
- 日志页支持“一键中文说明”模式（调用日志中文说明提示词）
- 账号管理支持主席一键重置部长/干事密码

## 7. 规则实现说明（当前版本）

- 出勤为硬门槛，不作为加分项
- 各阶段期数独立维护，并用于筛查与证书编号
- 激活学期是全局工作上下文（仪表盘、学员、作业、志愿、成绩均围绕激活学期）
- 成绩结果总览访问时会自动重算，确保与最新业务数据一致
- 证书编号会根据最新达标状态自动标注/撤销“(错误)”

## 8. 已做兼容处理

- 启动时会对历史数据库做兼容补充（如新增学期期数字段）
- 启动时会校验并升级学员唯一约束到“学期 + 学号 + 阶段”

## 9. 常见问题

- 启动报错找不到表/字段：先执行 `python init_db.py`，再启动。
- 在工作区根目录直接运行 `python app.py` 报文件不存在：请在项目目录 [app.py](app.py) 所在路径执行。
- AI 识别效果不稳定：优先规范输入为按行结构化文本，并检查提示词文件是否被误改。
